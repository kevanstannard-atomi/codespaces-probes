# web-hello

The "hello world" probe for GitHub Codespaces: a trivial web server whose page
is **generated by the devcontainer lifecycle** and served over a **forwarded
port**, so you can open it in your laptop browser and see a page served from a
container running in GitHub's cloud.

## What it proves

- Codespaces reads `.devcontainer/web-hello/devcontainer.json` and builds a
  container in the cloud.
- The `postCreateCommand` lifecycle hook runs — it generates `app/index.html`
  containing the container's hostname, user, and a build timestamp. That data
  can only come from inside the container, so the page **is** the proof the
  lifecycle ran.
- `forwardPorts: [8080]` makes the running server reachable from your laptop
  browser via a GitHub-issued forwarding URL.

## Files

| File | Runs where | Purpose |
|------|-----------|---------|
| `../../.devcontainer/web-hello/devcontainer.json` | GitHub (cloud) | container image + forwarded port + `postCreateCommand` |
| `postCreate.sh` | inside codespace | generates `app/index.html` (proof of lifecycle) |
| `serve.sh` | inside codespace | starts `python3 -m http.server 8080` in the background |
| `create.sh` | laptop | `gh codespace create` with `--devcontainer-path` |
| `verify.sh` | laptop | starts server, fetches page over ssh, asserts markers |
| `teardown.sh` | laptop | `gh codespace delete` |

## Running

One-time: the `gh` token needs the `codespace` scope.

```bash
gh auth refresh -h github.com -s codespace
```

Then:

```bash
cd ~/dev/codespaces-probes

CS=$(probes/web-hello/create.sh)   # prints the codespace name
probes/web-hello/verify.sh "$CS"   # PASS + prints the forwarded URL
# open the 8080 URL in your browser
probes/web-hello/teardown.sh "$CS" # delete when done
```
