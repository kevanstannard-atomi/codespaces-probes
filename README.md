# codespaces-probes

A collection of small, self-contained **probes** exploring GitHub Codespaces —
each one the simplest thing that visibly demonstrates a Codespaces capability.

This repo is **code only**. All research, planning, and findings live in the
Obsidian vault project `Projects/background/99-github-codespaces/`.

## How it's organised

One shared repo holds many probes. Codespaces discovers a devcontainer config at
`.devcontainer/<name>/devcontainer.json`, so **`.devcontainer/` is the catalogue
of probes** and you pick one at create time:

```bash
gh codespace create --repo kevanstannard-atomi/codespaces-probes \
  --devcontainer-path .devcontainer/<name>/devcontainer.json
```

You do **not** need a separate GitHub repo per probe.

```
codespaces-probes/
  .devcontainer/
    web-hello/devcontainer.json     # one folder per probe = the catalogue
  probes/
    web-hello/                      # that probe's code, scripts, and README
      app/                          # (generated at build time)
      postCreate.sh  serve.sh       # run inside the codespace
      create.sh  verify.sh  teardown.sh   # run from your laptop
      README.md
  AGENTS.md
```

## Probes

- [`probes/web-hello`](probes/web-hello/README.md) — "hello world": a page
  generated by the devcontainer lifecycle, served over a forwarded port, viewable
  in your browser.

## Prerequisites

```bash
gh auth refresh -h github.com -s codespace   # one-time: add the codespace scope
```
